Active Lives - Deterministic¶

Valuation Model¶

Documentation¶

class ActiveLivesValEMD(*, base_extract: pandas.core.frame.DataFrame, rider_rop_extract: pandas.core.frame.DataFrame, valuation_dt: pandas._libs.tslibs.timestamps.Timestamp, assumption_set: str, net_benefit_method: str, withdraw_table: str, interest_modifier: float = 1.0, incidence_modifier=1.0, withdraw_modifier=1.0)¶

Active lives deterministic valuation extract model.

This model takes an extract of policies and runs them through the respective Policy Models based on the COVERAGE_ID column (i.e., whether it is base policy or rider).

Parameters

base_extractDataFrame

The base active lives extract.

rider_rop_extractDataFrame

The ROP rider active lives extract.

valuation_dtTimestamp

The valuation date which reserves are based.

assumption_setstr

The assumption set to use for running the model. Options are :

  • stat

  • gaap

  • best-estimate

net_benefit_methodstr

The net benefit method. Options are :

  • NLP = Net level premium

  • PT1 = 1 year preliminary term

  • PT2 = 2 year preliminary term

withdraw_tablestr

Table name for withdraw rates.

Sensitivities

interest_modifierfloat

Interest rate modifier.

incidence_modifier

The incidence rate modifier.

withdraw_modifier

The withdraw rate modifier

Meta

model_versionstr

The model version generated by versioneer.

last_commitstr

The last git commit.

run_date_timeTimestamp

The run date and time.

Intermediates

recordsdict

The extract transformed to records.

Returns

projectedDataFrame

The projected reserves for the policyholders.

time_0DataFrame

The time 0 reserve for the policyholders

errorslist

Any errors captured.

Steps

  1. Create Records from Extract - Turn extract into a list of records for each row in extract.

  2. Run Records with Policy Models - Foreach record run through respective policy model based on COVERAGE_ID value.

  3. Get Time0 Values - Filter projected reserves frame down to time_0 reserve for each record.

To run the model call the run method which returns -

  • the projected reserve for each policy in the extract (projected),

  • the reserve as of the valuation date for each policy (time0), and

  • policies that error out when the model runs (run).

projected, time0, errors = model.run()

Note the time0 and projected have the same columns with time0 being a subset of the projected frame.
